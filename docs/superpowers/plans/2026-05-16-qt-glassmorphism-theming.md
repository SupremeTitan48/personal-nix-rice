# Qt Glassmorphism Theming Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire Qt apps into the matugen color pipeline via a custom Kvantum theme so they match the desktop's glassmorphism aesthetic and dynamic accent color.

**Architecture:** A Kvantum theme called `MatugenGlass` lives at `~/.config/Kvantum/MatugenGlass/MatugenGlass.kvconfig`. matugen regenerates this file on every wallpaper change using a new template. Kvantum's `composite=true`/`translucent_windows=true`/`blurring=true` settings make window backgrounds transparent; Hyprland's blur compositor renders the frosted-glass effect. No SVG required.

**Tech Stack:** NixOS, Home Manager, matugen (Material You color engine), Kvantum (Qt style engine), Hyprland

### Task 5: Add Qt env vars to UWSM environment

**Files:**
- Modify: `home/user/default.nix`

Home Manager's `qt` option writes `QT_STYLE_OVERRIDE` and `QT_QPA_PLATFORMTHEME` to `~/.nix-profile/etc/profile.d/hm-session-vars.sh`, which is sourced for login shells. The UWSM env file (`~/.config/uwsm/env-hyprland`) is read by UWSM before launching the session and propagates to D-Bus-activated services — apps launched from `.desktop` files rather than the terminal. Adding the vars here ensures every Qt app, however launched, uses Kvantum.

- [ ] **Step 1: Read the current file**

Read `home/user/default.nix` in full before editing.

- [ ] **Step 2: Add vars to the UWSM env block**

In the `xdg.configFile."uwsm/env-hyprland".text` block, add after `export QT_QPA_PLATFORM=wayland`:

```bash

    export QT_QPA_PLATFORMTHEME=kvantum
```

The full block should now end with:
```bash
    export QT_QPA_PLATFORM=wayland
```
