# Qt Glassmorphism Theming Design

**Date:** 2026-05-16
**Status:** Approved

## Goal

Make Qt apps (KeePassXC and any future Qt app) visually indistinguishable from GTK apps on this desktop. That means:

- Same dark surface colors as GTK
- Same accent color, derived from wallpaper via matugen
- Glassmorphism transparency — Hyprland's blur compositor handles the frosted-glass effect
- Colors update automatically when the wallpaper changes (requires Qt app restart to take effect)

## Approach

**Kvantum + matugen-generated kvconfig.**

Kvantum is the Qt style engine already wired up in `theme.nix`. A Kvantum "theme" is just a directory containing a `.kvconfig` file (SVG is optional). We create a theme called `MatugenGlass` whose kvconfig is regenerated by matugen on every wallpaper change — same pipeline as `gtk-colors.css`, `waybar-colors.css`, etc.

Transparency (`composite=true`, `translucent_windows=true`, `blurring=true`) in the kvconfig signals Kvantum to use transparent window backgrounds; Hyprland's blur layer renders the frosted glass. No SVG required.

The file contains only a comment and an empty module. It is not imported anywhere in `home/user/default.nix`. Delete it.

### 3. Update `modules/home/matugen.nix`

Three additions:

**a. Register template in config.toml:**
```toml
[[templates]]
input_path  = "~/.config/matugen/templates/kvantum-colors.kvconfig"
output_path = "~/.config/Kvantum/MatugenGlass/MatugenGlass.kvconfig"
```
Use the same `${config.xdg.configHome}` / `${config.home.homeDirectory}` interpolation as existing entries.

**b. Deploy template source file:**
```nix
xdg.configFile."matugen/templates/kvantum-colors.kvconfig".source =
  "${templateDir}/kvantum-colors.kvconfig";
```

**c. First-boot stub** in `home.activation.matugenInit`:
Create `~/.config/Kvantum/MatugenGlass/` with `mkdir -p` and write a stub `MatugenGlass.kvconfig` using the fallback blue accent (`#5b9cf6`) and dark surface (`#181c24`) — same fallback values used by every other stub in the block. Only write if the file doesn't already exist.

### 4. Update `modules/home/theme.nix`

**a. Remove unused packages** from `home.packages`:
- `libsForQt5.qt5ct` — GUI configurator, not needed; Nix manages the config
- `qt6Packages.qt6ct` — same

**b. Fix `qt.style.name`:** Change `"kvantum-dark"` → `"kvantum"`. Kvantum reads the active theme from its own config file; the style name just selects the Kvantum plugin, not a variant.

**c. Write the active-theme selector** as a static config file:
```nix
xdg.configFile."Kvantum/kvantum.kvconfig".text = ''
  [General]
  theme=MatugenGlass
'';
```
This is a static file (always points to `MatugenGlass`) so it belongs in `theme.nix` as an `xdg.configFile`, not in the matugen pipeline.

### 5. Update `home/user/default.nix` — UWSM env

Add to the `uwsm/env-hyprland` block:
```bash
export QT_STYLE_OVERRIDE=kvantum
export QT_QPA_PLATFORMTHEME=kvantum
```

Home Manager's `qt` option writes these to `~/.nix-profile/etc/profile.d/hm-session-vars.sh`, which is sourced for login shells. The UWSM env file ensures D-Bus-activated Qt apps (launched from `.desktop` files, not the terminal) also receive these vars.

## What Does NOT Change

- `change-wallpaper.sh` — no change needed. matugen already writes all templates including the new kvconfig. Qt apps pick up new colors on next restart.
- `modules/home/matugen/templates/` — add one file, everything else stays
- The Kvantum package selection in `theme.nix` — `libsForQt5.qtstyleplugin-kvantum` and `kdePackages.qtstyleplugin-kvantum` stay

## Runtime Behavior

1. **First login:** `home.activation.matugenInit` runs, generates stub `MatugenGlass.kvconfig` (or runs matugen from wallpaper if it exists). `kvantum.kvconfig` points to `MatugenGlass`. Qt apps open with fallback accent or wallpaper-derived accent.
2. **Wallpaper change:** `change-wallpaper.sh` calls `matugen image ...` which regenerates all templates including `MatugenGlass.kvconfig`. GTK apps reload instantly. Qt apps (e.g. KeePassXC) need to be restarted to pick up new colors — accepted limitation.
3. **Glassmorphism:** Kvantum's `translucent_windows=true` + `blurring=true` makes window backgrounds transparent. Hyprland's existing blur rules render the frosted-glass effect — no additional Hyprland config needed.
